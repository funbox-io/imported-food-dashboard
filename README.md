# 🌐 수입식품 실시간 원산지 모니터링 대시보드

식약처·관세청 **수입식품 검사연보** 공개 통계 기반의 실시간 통관 현황 모니터링 대시보드입니다.

**라이브 데모:** https://funbox-io.github.io/FunboxFood/

## 파일 구조

| 파일 | 역할 |
|---|---|
| `index.html` | 마크업 (KPI · 필터 · 원산지 순위 · 추이 차트 · 실시간 표 · 모달) |
| `style.css` | 다크 테마 전체 스타일 (IBM Plex Sans KR) |
| `data.js` | 검사연보 집계 통계 + 데모 샘플 데이터 풀 · 레코드 생성기 |
| `app.js` | 기능 로직 — 필터 · 렌더 · 드릴다운 · 모달 · CSV · 실시간 폴링 |

## 주요 기능

- **실시간 수집(폴링)** — 5초 주기 신규 통관 레코드 수집, 카운트다운·정지/재개, 파이프라인 로그
- **KPI** — 연간 수입신고 건수 · 원산지 국가 수 · 수입 중량/금액 · 세션 부적합(라이브)
- **필터** — 조회 기간 · 통합 검색(디바운스) · 원산지 · 통관 상태
- **원산지 국가별 현황** — 수집 데이터 기준 비중 % 실시간 재계산·재정렬, 국가 클릭 시 품목/업체 드릴다운(폴링 자동 갱신)
- **일별 수집 추이 차트** — 최근 14일 라인 차트(수집/부적합), 크로스헤어 hover 툴팁
- **신고 상세 모달** — 행 클릭 시 전체 필드 + 국내 유통/구매처(1~3곳, 주소)
- **구매자 상세 모달** — 구매자명(↗) 클릭 시 소재지·사업자정보·수입 이력·거래 유통처 전체
- **CSV 내보내기** — 현재 필터 결과 다운로드 (BOM 포함, Excel 한글 호환)

## 실행

정적 사이트라 빌드 없이 바로 열 수 있습니다:

```bash
python -m http.server 8000
# http://localhost:8000
```

## 실데이터 연동 (GitHub Actions 파이프라인)

기본은 데모 모드이며, 아래 설정 시 **식약처 수입식품 회수·판매중지(부적합) 실데이터**로 자동 전환됩니다.

```
GitHub Actions (매시간 cron)
  → data.go.kr 식약처 API 호출 (키는 Secrets — 소스에 노출 안 됨)
  → data/latest.json 생성·커밋
  → 대시보드가 60초마다 fetch → 신규 건 실시간 표시
```

**설정 방법**

1. [data.go.kr](https://www.data.go.kr) 가입 → **"식품의약품안전처_수입식품 회수판매중지 제품 정보"** ([바로가기](https://www.data.go.kr/data/15095378/openapi.do)) 활용신청 → 서비스 키 발급
2. 저장소 **Settings → Secrets and variables → Actions**에 등록:
   - `MFDS_API_KEY` (필수): 발급받은 서비스 키 (Encoding 키 권장)
   - `MFDS_API_URL` (선택): 활용신청 상세페이지의 정확한 요청주소(오퍼레이션 포함) — 기본값과 다를 경우만
3. **Actions 탭 → "Update import food data" → Run workflow**로 첫 실행 (이후 매시간 자동)
4. `data/latest.json`이 생성되면 대시보드가 자동으로 "● 실데이터" 모드로 전환됩니다

> 참고: 정부 공개 API는 초단위 실시간 스트림이 아니라 주기 갱신 데이터입니다.
> 회수·판매중지 API 특성상 실데이터 모드의 레코드는 전부 "부적합" 건입니다.

## 데이터 고지

> ⚠ 상단 KPI·원산지·품목군 통계는 공개 검사연보 기반 집계이며,
> 실시간 표의 기업·구매자·통관 내역은 **가상의 데모 샘플**로 실제 통관 기록이 아닙니다.
> 공식 데이터는 [수입식품정보마루](https://impfood.mfds.go.kr)에서 확인하세요.
